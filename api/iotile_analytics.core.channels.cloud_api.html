iotile_analytics.core.channels.cloud_api module¶

A wrapper around the iotile.cloud API to fetch data diretcly.

Summary¶

Classes:

IOTileCloudChannel An AnalysisGroupChannel that fetches data from IOTile.cloud

class iotile_analytics.core.channels.cloud_api.IOTileCloudChannel(cloud_id, domain, include_system=False)[source]¶

Bases: iotile_analytics.core.channels.channel.AnalysisGroupChannel

An AnalysisGroupChannel that fetches data from IOTile.cloud

Parameters:
  • domain (str) – The domain of the iotile.cloud instance that you want to fetch data from.
  • cloud_id (str) – The slug or id of the object to be enumerated to create this cloud channel. If you do not want to enumerate any object, you can pass None here.
  • include_system (bool) – Include hidden system streams when query device objects. This defaults to False.
list_streams()[source]¶

Return a list of all streams.

This is equivalent to the IOTile.cloud API method /api/v1/stream/

for datablocks and projects. For devices, this calls the api: /api/v1/device/<cloud_id>/extra/

to also get information on system streams that do not show up in the normal stream API.

Returns:
A list of dictionaries, one for each
stream that should be part of this analysis group.
Return type:list(dict)
