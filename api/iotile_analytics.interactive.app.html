iotile_analytics.interactive.app module¶

Default implementation for a bokeh application that can show itself in a server or notebook.

Summary¶

Classes:

AnalyticsApplication Base class for Bokeh applications built from iotile_analytics objects.
AnalyticsObject Base class for all objects that can be added to an AnalyticsApplication.

Reference¶

class iotile_analytics.interactive.app.AnalyticsObject[source]¶

Bases: object

Base class for all objects that can be added to an AnalyticsApplication.

These objects must have a root property that returns a Model or LayoutDOM object that corresponds to the root object that should be added to the application.

Bases: bokeh.application.application.Application

Base class for Bokeh applications built from iotile_analytics objects.

iotile_analytics.interactive.app.show(analytics_obj, interactive=True, **kwargs)[source]¶

Show an application or analytics object in a notebook.

This function will ultimately call bokeh.io.show() to actually show the object and with either wrap it inside an AnalyticsApplication object in order to have python callbacks work (if interactive==True, which is the default behavior) or it will just show the root Model objects without a bokeh server if interactive==False.
